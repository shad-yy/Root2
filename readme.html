SMART Live TV

Project Overview

SMART Live TV is a comprehensive sports streaming and information website that provides live sports updates, match schedules, news, and analytics for various sports including Football, UFC/MMA, Formula 1, and Boxing. The platform is designed with a modern, responsive interface that works seamlessly across desktop, tablet, and mobile devices.

Setup Instructions

Prerequisites

  • A modern web browser (Chrome, Firefox, Safari, Edge)
  • Basic knowledge of HTML, CSS, and JavaScript
  • A code editor (VS Code, Sublime Text, etc.)
  • A local development server (optional but recommended)

Installation

  1. Clone the repository or download the source code:

    git clone https://github.com/yourusername/smart-live-tv.git

  2. Navigate to the project directory:

    cd smart-live-tv

  3. Start a local development server:

    • For VS Code users, install the "Live Server" extension and click "Go Live"
    • For command line, you can use Python's built-in server: 
      python -m http.server
    • Or use Node.js with a package like http-server: 
      npx http-server

  4. Open your browser and navigate to http://localhost:8080 (or whatever port your server is using)

Configuration

The website's behavior can be customized by modifying the config.js file:

const CONFIG = {
  API: {
    BASE_URL: 'https://api.smartlivetv.net/v1',
    TIMEOUT: 10000,
    RETRY_ATTEMPTS: 3,
    CACHE_DURATION: 300000 // 5 minutes in milliseconds
  },
  FEATURES: {
    ENABLE_DARK_MODE: true,
    ENABLE_NOTIFICATIONS: true,
    SHOW_LIVE_INDICATOR: true
  },
  // ... additional configuration options
};

JavaScript Architecture

SMART Live TV follows a service-based architecture with modular JavaScript files:

Core Services

  1. config.js: Central configuration for the entire application
  2. error-handler.js: Handles errors across the application with consistent user feedback
  3. api-service.js: Manages all API calls with caching, retries, and error handling
  4. blog-service.js: Handles blog-related functionality like fetching posts and rendering
  5. reminder-service.js: Manages user reminders for upcoming sports events

Page-Specific Scripts

Each page has its own JavaScript file that imports the necessary services and handles page-specific logic:

  • index.js: Home page functionality
  • football.js: Football page functionality
  • ufc.js: UFC/MMA page functionality
  • f1.js: Formula 1 page functionality
  • boxing.js: Boxing page functionality
  • etc.

Module Pattern

All JavaScript files use ES modules for better organization and dependency management:

// Importing services
import CONFIG from './config.js';
import ApiService from './api-service.js';
import ErrorHandler from './error-handler.js';

// Feature implementation
function initPage() {
  // ... code
}

// Export functions that need to be accessible globally
window.functionName = functionName;

Page Structure

All pages follow a consistent structure:

  1. Head Section: Contains meta tags, title, CSS links, and Open Graph data
  2. Header: Main navigation and site logo
  3. Mobile Menu: Responsive navigation for smaller screens

  5. Footer: Site information, links, and copyright

API Integration

The api-service.js file handles all API communication:

Features

  • Caching: Reduces unnecessary API calls
  • Auto-Retry: Automatically retries failed requests
  • Error Handling: Consistent error handling
  • Response Normalization: Standardizes API responses

Usage Example

// Get upcoming matches
try {
  const matches = await ApiService.get('/football/matches/upcoming');
  renderMatches(matches);
} catch (error) {
  ErrorHandler.showError('Failed to load upcoming matches', error);
}

Error Handling

The error-handler.js service provides consistent error handling:

Features

  • Centralized Logging: All errors are logged in one place
  • User-Friendly Messages: Translates technical errors to user-friendly messages
  • Automatic Recovery: Provides retry options when appropriate
  • Error Tracking: Tracks unique errors to avoid repetitive notifications

Usage Example

try {
  // Code that might fail
} catch (error) {
  ErrorHandler.logError('operation_name', error);
  ErrorHandler.showError('User-friendly message');
}

Performance Optimization

The website incorporates several performance optimizations:

  1. Lazy Loading: Images are loaded only when needed
  2. API Caching: Prevents unnecessary API calls
  3. Proper Event Delegation: Minimizes event listeners
  4. CSS Organization: Efficient CSS selectors
  5. Throttling and Debouncing: For scroll and resize events

Deployment

Development Environment

For local development, use any static file server as described in the Setup Instructions.

Production Deployment

For production, we recommend:

  1. Static Site Hosting:
    • Netlify
    • Vercel
    • GitHub Pages
    • AWS S3 + CloudFront
  2. Build Process (if adding a build step in the future):

    • Optimize images

Example Netlify Deployment

  1. Create a netlify.toml file:

    [build]
  publish = "/"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

  2. Deploy to Netlify:

    • Connect your GitHub repository to Netlify
    • Configure build settings
    • Deploy
